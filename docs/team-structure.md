# AI金融信息推送Chrome插件 - Agent团队结构（简化版）

> **文档创建日期**: 2026-03-06
> **最后更新**: 2026-03-08（简化版）

---

## 概述

本文档定义了AI金融信息推送Chrome插件项目的Agent团队结构和角色分工。团队采用**7人协作模式**（简化版），每位Agent负责特定领域，确保工作无重叠。

**重要变更**: 根据简化版需求，已移除AI集成工程师角色，AI预测功能暂不开发。

---

## Agent角色分配

### 1. 产品经理

**Agent身份**: `finance-news-pm`

**核心职责**:
- 需求细化与澄清
- 功能优先级排序
- 验收标准(DoD)定义
- 用户故事编写

**主要任务**:

2. 制定MVP功能清单（移除AI预测）
3. 定义每个功能的验收标准
4. 规划迭代周期和里程碑
5. 风险评估和缓解措施

**产出物**:
- 简化版需求规格说明书
- 用户故事清单
- 验收标准文档
- 项目里程碑计划

**依赖关系**:

- 所有开发人员需要验收标准来完成任务

## 工作流程

### 并行工作阶段

```
阶段1: 基础设施搭建 (并行)
├── 产品经理: 需求细化（简化版）
├── UI/UX设计师: 设计规范（移除AI组件）
└── 架构师: 项目脚手架

阶段2: 核心功能开发 (并行)
├── 后端开发工程师: 数据抓取 + Chrome Storage
├── 前端开发工程师A: Content Script
├── 前端开发工程师B: Popup页面
└── 测试开发工程师: 测试框架

阶段3: 集成测试 (协作)
└── 所有agent: 集成联调
```

### 关键交接点

| 交接方 | 接收方 | 交接内容 |
|--------|--------|----------|
| 产品经理 | 架构师 | 简化版需求规格说明书 |
| UI/UX设计师 | 前端开发A/B | 简化版设计规范文档 |
| 架构师 | 所有开发 | 项目脚手架、环境配置 |
| 后端开发工程师 | 前端开发A/B | 数据接口（新闻数据） |
| 所有开发 | 测试开发工程师 | 待测试代码 |

## 通用协作

### 工作交接规范

1. **文档化** - 所有关键产出物必须形成文档
2. **接口定义** - 前后端对接前必须先定义接口
3. **代码审查** - 核心模块完成后进行code review
4. **集成测试** - 每个功能模块完成后进行集成测试

### 冲突解决

- 职责重叠时，优先参考本文档中的角色定义
- 技术方案冲突由架构师裁决
- 需求冲突由产品经理裁决
