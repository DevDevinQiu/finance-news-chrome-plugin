# AI金融信息推送Chrome插件 - Agent团队结构

> 文档创建日期: 2026-03-06

## 概述

本文档定义了AI金融信息推送Chrome插件项目的Agent团队结构和角色分工。团队采用8人协作模式，每位Agent负责特定领域，确保工作无重叠。

---

## Agent角色分配

### 1. 产品经理

**Agent身份**: `finance-news-pm`

**核心职责**:
- 需求细化与澄清
- 功能优先级排序
- 验收标准(DoD)定义
- 用户故事编写

**主要任务**:
1. 编写详细的需求规格说明
2. 制定MVP功能清单
3. 定义每个功能的验收标准
4. 规划迭代周期和里程碑
5. 风险评估和缓解措施

**产出物**:
- 需求规格说明书
- 用户故事清单
- 验收标准文档
- 项目里程碑计划

**依赖关系**:
- 架构师需要需求文档进行技术选型
- 所有开发人员需要验收标准来完成任务

### 3. 架构师

**Agent身份**: `finance-news-architect`

**核心职责**:
- 项目脚手架初始化
- 开发环境配置
- 构建配置和打包脚本
- 目录结构规划

5. 编写构建和打包脚本
6. 规划项目目录结构
7. 编写开发环境配置文档

**产出物**:
- 项目脚手架（package.json, tsconfig.json等）
- ESLint/Prettier配置
- Git hooks配置
- 构建脚本
- README.md（开发环境说明）

**依赖关系**:
- 产品经理提供需求文档
- UI/UX设计师提供设计规范
- 为所有开发人员提供基础开发环境

### 6. 后端开发工程师

**Agent身份**: `finance-news-backend`

**核心职责**:
- 数据抓取引擎

**主要任务**:

3. 实现SQLite数据库初始化和CRUD操作

5. 实现请求重试和错误处理
6. 实现chrome.storage API集成
7. 实现新闻分类本地规则引擎

**产出物**:
- `src/background/fetchers/` 数据抓取模块
- `src/background/database/` 数据库操作模块
- `src/background/utils/` 工具函数
- 数据库schema定义

**依赖关系**:
- 架构师提供项目脚手架
- AI集成工程师使用数据库存储预测结果

## 工作流程

### 并行工作阶段

```
阶段1: 基础设施搭建 (并行)
├── 产品经理: 需求细化
├── UI/UX设计师: 设计规范
└── 架构师: 项目脚手架

阶段2: 核心功能开发 (并行)
├── 后端开发工程师: 数据抓取 + 数据库
├── AI集成工程师: AI集成
├── 前端开发工程师A: Content Script
├── 前端开发工程师B: Popup页面
└── 测试开发工程师: 测试框架

阶段3: 集成测试 (协作)
└── 所有agent: 集成联调
```

### 关键交接点

| 交接方 | 接收方 | 交接内容 |
|--------|--------|----------|
| 产品经理 | 架构师 | 需求规格说明书 |
| UI/UX设计师 | 前端开发A/B | 设计规范文档 |
| 架构师 | 所有开发 | 项目脚手架、环境配置 |
| 后端开发工程师 | 前端开发A/B/AI | 数据接口 |
| AI集成工程师 | 前端开发A | 预测数据接口 |
| 所有开发 | 测试开发工程师 | 待测试代码 |

## 沟通协作

### 工作交接规范

1. **文档化** - 所有关键产出物必须形成文档
2. **接口定义** - 前后端对接前必须先定义接口
3. **代码审查** - 核心模块完成后进行code review
4. **集成测试** - 每个功能模块完成后进行集成测试

### 冲突解决

- 职责重叠时，优先参考本文档中的角色定义
- 技术方案冲突由架构师裁决
- 需求冲突由产品经理裁决
