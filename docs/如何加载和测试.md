# 如何加载和测试 Chrome 扩展

## 1. 构建扩展

确保项目已构建：
```bash
yarn build
```

构建成功后，`dist/` 目录包含扩展所需的所有文件。

## 2. 加载到 Chrome 浏览器

### 方法一：开发者模式加载
1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions/`
3. 开启右上角的「开发者模式」开关
4. 点击「加载已解压的扩展程序」
5. 选择项目目录：`dist/`
6. 扩展将出现在扩展列表中

### 方法二：直接拖放
1. 打开 Chrome 浏览器
2. 打开文件管理器，找到 `dist/` 文件夹
3. 将 `dist/` 文件夹拖放到 Chrome 扩展页面
4. 扩展将自动安装

## 3. 测试功能

### 3.1 测试 Content Script 面板
1. 访问任何网站（例如 https://www.google.com）
2. 查看浏览器右侧是否出现可展开/收起的按钮
3. 点击按钮，查看金融面板是否展开
4. 查看面板中是否显示新闻列表

### 3.2 测试设置按钮功能
1. 访问任何网站
2. 查看面板右上角的设置按钮 ⚙️
3. 点击设置按钮
4. 验证是否正确打开 Options 页面（`chrome-extension://xxx/src/options/index.html`）
5. 检查是否有 ERR_BLOCKED_BY_CLIENT 错误
6. 如果有错误，请打开 DevTools 查看控制台日志
7. 或者右键点击扩展图标，选择「选项」
8. 查看 Options 页面是否正确打开
9. 测试 Tab 导航切换
10. 测试配置保存功能

### 3.3 测试 Popup 页面
1. 点击浏览器工具栏中的扩展图标
2. 查看 Popup 页面是否正确打开
3. 检查设置选项（新闻源切换、刷新间隔、最大数量）
4. 点击「立即刷新」按钮，查看统计数据更新

### 3.4 测试 Background 自动刷新
1. 打开 Chrome DevTools（按 F12）
2. 切换到「Service Worker」标签
3. 查看 Background Service Worker 日志
4. 等待几分钟，查看是否有「Refresh alarm triggered」日志
5. 查看日志中是否显示「Fetched X news items」

### 3.5 验证数据存储
1. 打开 Chrome DevTools（按 F12）
2. 切换到「Application」标签
3. 在左侧找到「Storage」→「Local Storage」
4. 查看是否有 `newsList`、`settings`、`stats` 数据键
5. 验证数据是否正确更新

## 4. 常见问题

### 问题：面板未显示
**可能原因**：
- Content Script 未注入
- 样式被遮挡
- JavaScript 错误

**解决方法**：
1. 检查 Service Worker 日志是否有错误
2. 检查网站是否有 CSP 策略限制
3. 尝试在不同网站（如空白页）加载
4. 刷新扩展：在 `chrome://extensions/` 页面点击扩展的刷新按钮

### 问题：Popup/Options 页面空白
**可能原因**：
- React 组件渲染失败
- 资源加载问题

**解决方法**：
1. 打开 Console 查看错误
2. 检查是否缺少 React/Vite 构建文件
3. 重新构建扩展：`yarn build`
4. 在 `chrome://extensions/` 页面点击扩展的刷新按钮

### 问题：自动刷新不工作
**可能原因**：
- Alarms API 未正确设置
- 间隔配置过短

**解决方法**：
1. 检查 settings 中的 refreshInterval 值
2. 在 Options 页面调整间隔（建议 1 分钟测试）
3. 重启扩展：在 `chrome://extensions/` 页面点击扩展的刷新按钮

## 5. 调试模式

### 查看 Service Worker 日志
1. 右键点击扩展图标 →「检查视图」
2. 查看后台日志输出

### 查看 Content Script 日志
1. 访问网站
2. 按 F12 打开 DevTools
3. 查看 Console 日志

## 6. 开发模式

### 热重载
修改代码后：
1. 重新构建：`yarn build`
2. 在 `chrome://extensions/` 页面点击扩展的刷新按钮

### 自动重载
在开发过程中，可以使用 CRXJS 的开发模式：
```bash
yarn dev
```
这会启动 Vite 开发服务器，扩展会自动重载。

## 7. 清除数据

如需清除所有存储数据：
1. 访问 `chrome://extensions/`
2. 点击扩展的「移除」按钮
3. 重新加载扩展

或在 DevTools → Application → Storage → Clear

---

*注意*：初次加载时，扩展需要一些时间初始化数据和设置。
